Já há algum tempo as empresas de desenvolvimento de software têm vivenciado o que se tem chamado de aflição crônica (conjunto de problemas que são encontrados no desenvolvimento de sistemas). Frente à grande demanda e a forte competitividade existente no mercado atual, tais problemas podem inviabilizar a sobrevivência de uma organização.
Assim, o alcance de alguns objetivos, tais como aumento de produtividade e redução de custos, entre outros, tem sido apresentado como bastante importante para o sucesso de uma empresa. 
O reúso sistemático de componentes de software tem colaborado de forma significativa para o alcance desses objetivos. Baseado no desenvolvimento de sistemas a partir de componentes pré-existentes, identificando semelhanças entre o novo sistema e esses componentes, tal modelo permite reduções significativas de custo e tempo de desenvolvimento.
